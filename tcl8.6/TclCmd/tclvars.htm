DESCRIPTION

The following global variables are created and managed automatically by the Tcl library. 以下のグローバル変数は、Tclライブラリによって自動的に作成および 管理されます。 Except where noted below, these variables should normally be treated as read-only by application-specific code and by users. 以下に記載がない限り、これらの変数はアプリケーション固有のコードや ユーザーによって通常は読み取り専用として扱われるべきです。
auto_path
If set, then it must contain a valid Tcl list giving directories to search during auto-load operations (including for package index files when using the default package unknown handler). 設定されている場合、有効なTclリストが含まれている必要があり、 自動ロード操作中に検索するディレクトリを指定します (デフォルトの package unknown ハンドラーを使用する場合のパッケージインデックスファイルを含む）。 This variable is initialized during startup to contain, in order: the directories listed in the TCLLIBPATH environment variable, the directory named by the tcl_library global variable, the parent directory of tcl_library, the directories listed in the tcl_pkgPath variable. この変数は起動時に初期化され、順に TCLLIBPATH 環境変数に リストされたディレクトリ、tcl_library グローバル変数で 指定されたディレクトリ、tcl_library の親ディレクトリ、 tcl_pkgPath 変数にリストされたディレクトリが含まれます。 Additional locations to look for files and package indices should normally be added to this variable using lappend. ファイルやパッケージインデックスを検索するための追加の場所は、 通常 lappend を使用して この変数に追加する必要があります。

Additional variables relating to package management exist. パッケージ管理に関連する追加の変数が存在します。 More details are listed in the VARIABLES section of the library manual page. 詳細は library マニュアル ページの VARIABLES セクションに記載されています。
env
This variable is maintained by Tcl as an array whose elements are the environment variables for the process. この変数はプロセスの環境変数要素として Tcl によって 配列として管理されます 。 Reading an element will return the value of the corresponding environment variable. 要素を読み取ると、対応する環境変数の値が返されます。 Setting an element of the array will modify the corresponding environment variable or create a new one if it does not already exist. 配列の要素を設定すると、対応する環境変数が変更され、 既に存在しない場合は新しい変数が作成されます。 Unsetting an element of env will remove the corresponding environment variable. env の要素を解除すると、対応する環境変数が削除されます。 Changes to the env array will affect the environment passed to children by commands like exec. env 配列の変更は、 exec のようなコマンドによって 子プロセスに渡される環境に影響します。 If the entire env array is unset then Tcl will stop monitoring env accesses and will not update environment variables. env 配列全体が解除された場合、Tcl は env へのアクセスを監視しなくなり、 環境変数を更新しません。

Under Windows, the environment variables PATH and COMSPEC in any capitalization are converted automatically to upper case. Windows では、PATH および COMSPEC 環境変数は、どのような大文字小文字でも 自動的に大文字に変換されます。 For instance, the PATH variable could be exported by the operating system as "path", "Path", "PaTh", etc., causing otherwise simple Tcl code to have to support many special cases. 例えば、PATH 変数はオペレーティングシステムによって "path"、"Path"、"PaTh" などとしてエクスポートされ、 単純な Tcl コードが多くの特別なケースをサポート しなければならない原因となります。 All other environment variables inherited by Tcl are left unmodified. Tcl が継承する他のすべての環境変数は変更されません。 Setting an env array variable to blank is the same as unsetting it as this is the behavior of the underlying Windows OS. env 配列変数を空白に設定することは、 それを解除することと同じであり、これは基礎となる Windows OS の動作です。 It should be noted that relying on an existing and empty environment variable will not work on Windows and is discouraged for cross-platform usage. 既存の空の環境変数に依存することは Windows では 機能せず、クロスプラットフォームでの使用は推奨されません。
